# VIVID-Det 프레임워크 소개

**VIVID-Det: Video-Integrated Vision for Intelligent Detection**

RF-DETR 기반 소형 객체 검출 특화 프레임워크

---

## 1. VIVID-Det이란

VIVID-Det은 RF-DETR이라는 실시간 객체 검출 모델 위에 소형 객체 검출 성능을 끌어올리기 위한 5개 커스텀 모듈과 1개 추론 전략을 추가한 프레임워크다.

RF-DETR은 COCO 데이터셋에서 실시간 검출기 최초로 60 AP를 달성한 모델이지만, 소형 객체(AP_S)에서는 여전히 대형 객체 대비 성능 격차가 크다. VIVID-Det은 이 격차를 RF-DETR의 구조를 변경하지 않으면서 줄이는 것을 목표로 한다.

핵심 설계 원칙은 **NAS-Safe**다. RF-DETR은 Neural Architecture Search(NAS)로 최적화된 구조를 가지고 있으므로, 이 구조를 건드리면 NAS가 찾은 최적점에서 이탈한다. VIVID-Det의 모든 모듈은 NAS가 관리하지 않는 파이프라인 간극에서만 동작한다.

---

## 2. RF-DETR-M과 RF-DETR-L 기본 스펙

VIVID-Det은 RF-DETR의 두 가지 변형 위에서 동작한다. 동일한 DINOv2 ViT-S/14 백본을 사용하지만 입력 해상도가 다르며, 이 차이가 모든 모듈의 내부 수치에 영향을 준다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 입력 해상도 | 576 x 576 | 704 x 704 |
| ViT 패치 크기 | 14 x 14 | 14 x 14 |
| ViT 토큰 수 | 41 x 41 = 1,681 | 50 x 50 = 2,500 |
| 백본 출력 차원 | 384 | 384 |
| Baseline AP (COCO) | 54.7 | 56.5 |
| Baseline AP_S (COCO) | 36.1 | (재현 예정) |
| Baseline 파라미터 | 33.7M | 33.9M |
| Baseline 레이턴시 (T4) | 4.4ms | 6.8ms |

M은 속도 우선, L은 정밀도 우선이다. 군경계 데이터(1920x1080) 기준으로 동일한 Person 객체(원본 ~35x55px)가 M에서는 ~1.6 패치, L에서는 ~2.4 패치로 표현된다. 둘 다 소형 객체에는 부족하지만 L이 50% 더 많은 패치를 확보한다.

## 3. 명칭 정리

### 프레임워크

| 약칭 | 풀네임 | 설명 |
|------|--------|------|
| VIVID-Det | Video-Integrated Vision for Intelligent Detection | 전체 프레임워크 |

### 학습 모듈 (5개)

| 순서 | 약칭 | 풀네임 | 위치 | 역할 |
|------|------|--------|------|------|
| 1 | TFCM | Temporal Feature Correspondence Module | Backbone → SSFA 사이 | 비디오에서 과거 프레임 정보를 현재 프레임에 융합 |

| 4 | SAWM | Scale-Aware Wasserstein Matching | Hungarian Matching | 소형 객체에 유리한 거리 척도(NWD)로 매칭 비용 보완 |
| 5 | SQALB | Scale-Quality Adaptive Loss Balancer | Loss 계산 | 객체 크기와 품질에 따라 손실 함수 비율을 자동 조정 |

|------|--------|------|
| SGAR | SOPM-Guided Adaptive Resolution | 소형 객체가 있을 법한 영역만 골라서 확대 재추론 |

| 약칭 | 풀네임 | 역할 |
|------|--------|------|
| SOPM | Small Object Possibility Map | SSFA가 생성하는 소형 객체 가능성 맵. TFCM, SAQG, SGAR이 공유 |

## 4. 각 모듈이 하는 일

### 4.1 SSFA: 고해상도 세부 정보 주입

RF-DETR의 DINOv2 ViT 백본은 14x14 패치 단위로 이미지를 처리한다. 패치 크기가 고정이므로 입력 해상도가 커지면 토큰 수가 증가하지만, 소형 객체는 여전히 소수의 토큰으로만 표현된다.

SSFA는 별도의 경량 CNN(stride-8)을 통해 ViT보다 촘촘한 특징맵을 추출하고, SOPM 값이 높은 상위 25%의 토큰만 골라서 cross-attention의 K/V로 사용한다. 이 토큰들은 공간적으로 연속된 영역이 아니라 SOPM 값 순서대로 선택된 것이므로 이미지 전역에 흩어져 있을 수 있다. 다만 cross-attention의 Q-K 유사도 메커니즘이 의미적으로 관련 없는 토큰의 기여를 자동으로 억제하므로, 엉뚱한 배경 토큰이 섞여 있어도 최종 출력에 미치는 영향은 제한적이다. 부산물로 SOPM(Small Object Possibility Map)을 생성하며, 이 SOPM이 프레임워크 전체를 관통하는 통합 신호가 된다.

> TIP: Top-K 비율 25%는 현재 고정값이다. {10%, 15%, 25%, 50%} 범위에서 recall과 연산량의 최적점을 탐색할 여지가 있다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| ViT 토큰 수 (Q) | 41 x 41 = 1,681 | 50 x 50 = 2,500 |
| CNN 특징맵 해상도 (K/V) | 72 x 72 = 5,184 | 88 x 88 = 7,744 |
| Top-K 25% 선택 후 | ~1,296 토큰 | ~1,936 토큰 |
| SOPM 해상도 | 72 x 72 | 88 x 88 |
| 추론 비용 | +1~3% | +1~3% |

L은 CNN 해상도가 더 높아(88x88 vs 72x72) 소형 객체의 세부 특징을 더 정밀하게 추출할 수 있다.

```
DINOv2 ViT 출력 (M: 41x41 / L: 50x50)
   +
CNN 출력 (M: 72x72 / L: 88x88)
   ↓ SOPM 값 상위 25% 토큰을 K/V로 선택
   ↓ cross-attention (Q=ViT 전체, K/V=선택 토큰)
향상된 특징 + SOPM 생성
```

### 4.2 TFCM: 과거 프레임에서 힌트 얻기

CCTV 영상에서 소형 객체는 한 프레임에서는 거의 보이지 않지만, 여러 프레임을 모아보면 "여기 뭔가 있다"는 판단이 가능하다. TFCM은 최근 4프레임의 특징을 기억해두고, DINOv2 특징 공간에서 프레임 간 대응점을 찾아 현재 프레임의 소형 객체 특징을 보강한다.

대형 객체 영역까지 시간적 융합을 하면 오히려 성능이 떨어질 수 있다(Utrecht University 실험에서 확인됨). 그래서 SOPM(t-1)을 마스크로 사용하여 소형 객체 가능 영역에서만 시간적 융합을 수행한다.

이미지 모드(단일 프레임)에서는 완전히 bypass되어 추가 비용이 0이다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 대응 행렬 크기 | 1,681 x 1,681 | 2,500 x 2,500 |
| 대응 행렬 메모리 (4프레임) | 22.6 MB | 50.0 MB |
| TFCM-Full GFLOPs | ~17.4 | ~38.4 |
| TFCM-Sparse GFLOPs | ~2.2 | ~4.8 |
| 추론 비용 (비디오) | +5~8% | +8~12% |
| 권장 모드 | Full 또는 Sparse | Sparse (비용 절감) |

L은 토큰 수가 1.5배이므로 대응 행렬이 2.2배 커진다(N^2 복잡도). L에서는 SOPM 기반으로 상위 25% 토큰만 대응 계산하는 TFCM-Sparse를 기본 설정으로 권장한다.

```
비디오 모드:
  과거 4프레임 특징 (Memory Buffer)
    ↓ cosine similarity로 대응점 찾기
    ↓ softmax(τ=0.03 learnable)로 가중치 계산
    ↓ SOPM(t-1) 마스크: 소형 객체 영역만 융합
  현재 프레임 특징 보강

  M: 1,681 x 1,681 대응 행렬 (Full 가능)
  L: 2,500 x 2,500 → Sparse 권장 (625 x 2,500)

이미지 모드:
  bypass (비용 0%)
```

> TIP: 참조 프레임 수 4와 softmax temperature τ=0.03은 현재 고정값이다. {2, 4, 8} 프레임과 τ={0.01, 0.03, 0.05, 0.07} 범위에서 최적점을 탐색할 여지가 있다.

### 4.3 SAQG: 검출 쿼리를 소형 객체 쪽으로 모으기

RF-DETR의 디코더는 쿼리 토큰으로 객체를 찾는다. 기본적으로 이 쿼리들은 이미지 전체에 균일하게 분포하는데, 소형 객체가 밀집된 영역에 쿼리가 부족하면 검출을 놓칠 수 있다.

SAQG는 SOPM을 참고하여 소형 객체 가능성이 높은 영역에 더 많은 쿼리를 배치한다. 총 쿼리 수는 변하지 않으므로 NAS 구조에 영향을 주지 않는다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 쿼리 수 | NAS 결정값 | NAS 결정값 |
| SOPM 입력 해상도 | 72 x 72 | 88 x 88 |
| 추론 비용 | ~0% (무시 수준) | ~0% (무시 수준) |

SAQG는 모델 안에 항상 존재하여 매 forward pass마다 동작하지만, 효과는 객체의 패치 스케일에 따라 달라진다.

- **Pass 1** (전체 이미지): 극소형 객체가 M에서 ~1.6패치, L에서 ~2.4패치 수준이면 쿼리를 집중시킬 공간 자체가 없어 효과가 제한적이다. 다만 중형 이상 객체나 COCO 스케일(5+ 패치) 소형 객체에서는 유효하다.
- **Pass 2** (SGAR crop 이미지): crop으로 소형 객체가 21+ 패치로 확대되면 쿼리 재배치가 의미 있는 범위에 들어온다. crop 안에 소형 객체가 여러 개 밀집된 상황에서 특히 효과적이다.

즉, SAQG의 핵심 효과는 SGAR Pass 2에서 발휘된다.

### 4.4 SAWM: 소형 객체에 맞는 매칭 기준

DETR 계열 모델은 Hungarian Matching으로 예측과 GT를 1:1 매칭한다. 기본 매칭 비용은 L1 거리와 GIoU인데, 소형 객체에서는 1~2px 오차만 있어도 IoU가 급락하여 매칭이 불안정해진다.

SAWM은 Normalized Wasserstein Distance(NWD)를 매칭 비용에 추가한다. NWD는 bbox를 가우시안 분포로 모델링하여, 소형 객체에서도 위치 근접성을 안정적으로 평가한다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 소형 객체 패치 수 | ~1.6 | ~2.4 |
| 1px 오차 시 IoU 하락 | 극심 | 심각 (M보다 완화) |
| NWD 효과 | 핵심적 (패치 극소) | 유효 (패치 소) |
| 추론 비용 | 0% | 0% |

M에서 소형 객체의 패치 수가 더 적으므로 IoU 기반 매칭의 불안정성이 더 심하고, NWD의 개선 효과도 상대적으로 더 크다. 학습 전용 모듈로 추론 시에는 동작하지 않아 추론 비용이 0이다.

> TIP: NWD 매칭 비용의 가중치 λ_nwd는 현재 고정값이다. {2.0, 3.5, 5.0} 범위에서 AP_S와 AP_L의 trade-off 최적점을 탐색할 여지가 있다.

### 4.5 SQALB: 크기에 따라 손실 함수 자동 조정

소형 객체와 대형 객체에 동일한 손실 함수 비율을 적용하면, 대형 객체의 큰 gradient가 소형 객체의 gradient를 압도한다. SQALB는 5개 입력(면적, 신뢰도, IoU 품질, 디코더 레이어 번호, 매칭 비용)을 MLP에 넣어 NWD와 WIoU의 비율을 자동 결정한다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 면적 정규화 | 해상도 인식 (576 기준) | 해상도 인식 (704 기준) |
| area-bin EMA 구간 | small / medium / large | small / medium / large |
| 추론 비용 | 0% | 0% |

SQALB의 면적 입력은 해상도를 고려한 정규화된 값(normalized area log)을 사용하므로, M과 L에서 동일한 객체가 다른 면적 값을 가지더라도 일관된 밸런싱이 가능하다. 학습 전용 모듈로 추론 비용 0이다.

> TIP: area-bin EMA의 구간 경계(small/medium/large)와 decay rate는 현재 COCO 표준 기준 고정값이다. 군경계 데이터의 크기 분포에 맞춰 조정할 여지가 있다.

## 5. SOPM 통합 신호

VIVID-Det의 구조적 차별점은 SOPM이라는 하나의 신호가 여러 모듈을 유기적으로 연결한다는 것이다. 이것이 단순한 모듈 나열이 아닌 "프레임워크"인 이유다.

```
            ┌────────┐
            │  SSFA  │
            │ (생성)  │
            └───┬────┘
                │
        SOPM (M: 72x72 / L: 88x88)
                │
      ┌─────────┼─────────┐
      v         v         v
  ┌────────┐ ┌────────┐ ┌──────────┐
  │  TFCM  │ │  SAQG  │ │   SGAR   │
  │(마스크) │ │(배치)  │ │ (crop)    │
  └────────┘ └────────┘ └──────────┘

  TFCM: SOPM(t-1)로 소형 영역만 시간적 융합
  SAQG: SOPM으로 소형 밀집 영역에 쿼리 집중
  SGAR: SOPM으로 확대 재추론할 영역 선택
```

## 6. 추론 파이프라인: SGAR

SGAR은 학습과 무관하게, 학습 완료 모델의 추론 시에만 동작하는 전략이다.

### 6.1 문제

군경계 데이터(1920x1080)를 RF-DETR에 넣으면 Person이 극소수 패치로 축소된다.

| 항목 | RF-DETR-M (576x576) | RF-DETR-L (704x704) |
|------|---------------------|---------------------|
| Person 리사이즈 후 | ~10.5 x 16.5 px | ~12.8 x 20.2 px |
| ViT 패치 수 | ~1.6 | ~2.4 |
| mAP@50 (위치) | ~0.96 | (측정 예정) |
| mAP@50:95 (정밀도) | ~0.60 | (측정 예정) |
| 격차 | 36.2%p | (측정 예정) |

L이 M보다 50% 더 많은 패치를 확보하지만, 2.4 패치도 정밀한 bbox에는 여전히 부족하다. 두 모델 모두 SGAR의 해상도 증가가 필요하다.

### 6.2 해결: 2-Pass 적응적 확대

```
원본 이미지 (1920x1080)
     │
     v
Pass 1: M:576x576 / L:704x704 리사이즈 → 모델 추론
     │   (SAQG 동작하지만 극소형 객체에는 효과 제한적)
     │
     ├─→ 대형/중형 객체 검출 확정
     ├─→ SOPM 획득 (SSFA 부산물, 추가 비용 0)
     │
     v
SOPM에 소형 객체 영역이 있는가?
     │
     ├─ 없음 → Pass 1 결과 반환 (추가 비용 0)
     │
     └─ 있음 → crop 영역 선택 (최대 4개)
              │
              v
Pass 2: 각 crop → M:576x576 / L:704x704 리사이즈 → 동일 모델 재추론
              │   (소형 객체가 21+ 패치로 확대 → SAQG 효과 유효)
              │
              v
         결과 병합 (Soft-NMS로 중복 제거)
```

### 6.3 효과

400x400 crop으로 Person을 재추론할 때:

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| Pass 1 패치 수 | ~1.6 | ~2.4 |
| Pass 2 패치 수 (400x400 crop) | ~21.6 | ~32.3 |
| 배율 | 13.5x | 13.5x |

L은 Pass 2에서 32.3 패치를 확보하여 M의 21.6 패치보다 정밀한 bbox가 가능하다.

### 6.4 SAHI 대비 장점

| 항목 | SAHI | SGAR |
|------|------|------|
| crop 결정 | 고정 격자 (4~16 슬라이스) | SOPM 기반 적응적 |
| 소형 객체 없는 이미지 | 동일 비용 발생 | 추가 비용 0 (스킵) |
| 평균 추론 비용 | 4~9x | 1.3~1.8x |

> TIP: SGAR의 max_crops=4와 SOPM 이진화 임계값 τ_crop=0.3은 현재 고정값이다. crop 수 {1, 2, 4}와 τ_crop {0.1, 0.3, 0.5} 범위에서 정밀도-속도 최적점을 탐색할 여지가 있다.

## 8. 3단계 학습 전략

| Stage | 데이터 | 활성 모듈 | 목적 |
|-------|--------|----------|------|
| Stage 1 | 이미지 (군경계) | SSFA + SAQG + SAWM + SQALB | 공간 모듈 학습 |
| Stage 2 | 비디오 (연속 프레임) | + TFCM 추가 활성화 | 시간적 모듈 학습 |
| Stage 3 | (선택적) | 전체 | 공동 미세조정 |

M과 L은 동일한 코드베이스에서 독립 학습한다. M에서 확정된 하이퍼파라미터를 L 학습의 시작점으로 사용한다.

| 항목 | RF-DETR-M | RF-DETR-L |
|------|-----------|-----------|
| 물리 배치 | 16 | 4 |
| Gradient Accumulation | 1 | 4 |
| 유효 배치 | 16 | 16 |
| 예상 VRAM | ~22-24GB | ~28-30GB |

Stage 2에서 TFCM은 alpha = sigmoid(-10) ≈ 0에서 시작하여, 학습이 진행됨에 따라 점진적으로 시간적 융합 비율이 증가한다. 이렇게 하면 Stage 1에서 학습한 가중치를 교란하지 않으면서 안전하게 시간적 모듈을 통합할 수 있다.

## 11. 배포 환경별 구성

| 환경 | 모드 | SSFA/SAQG | SGAR | TFCM | M 레이턴시 | L 레이턴시 |
|------|------|-----------|------|------|----------|----------|
| Jetson Orin Nano 8GB | Speed | ON | OFF | OFF | ~5ms | (VRAM 부족) |
| AGX Orin 64GB | Balanced | ON | crop 2개 | Sparse | ~15ms | ~20ms |
| RTX 5090 32GB | Accuracy | ON | crop 4개 | M:Full / L:Sparse | ~8ms | ~12ms |

모든 모드에서 SSFA(+1~3%)와 SAQG(~0%)는 항상 활성이다. 또한 SAWM/SQALB는 학습 전용이지만, 학습 시 개선된 가중치가 모델에 반영되어 있으므로 모든 모드에서 간접적으로 효과가 유지된다. 즉, Speed 모드도 RF-DETR baseline과는 다르다.

Jetson Orin Nano(8GB)에서는 L 모델 자체가 VRAM 한계로 운용 어려울 수 있으며, M + Speed 모드가 적합하다. AGX Orin(64GB)에서는 M/L 모두 운용 가능하나 TFCM-Full은 L에서 메모리 부담이 크므로 Sparse를 권장한다.
